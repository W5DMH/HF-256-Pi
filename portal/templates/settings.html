Direwolf AX.25 Hub & Spoke

Direwolf provides AX.25 connected-mode packet radio. As a hub, Direwolf listens for incoming spoke connections on the configured channels simultaneously. As a spoke, Direwolf lets you connect to a hub via VHF AX.25 (9600 baud) or HF AX.25 (300 baud) using /connect <HUBCALL> in the Console. Configure one or both channels below. Leave audio card blank to disable that channel. Direwolf must be installed (apt install direwolf).

⬡ VHF Channel — Port 0 (9600 baud G3RUH)

DigiRig Mobile: select /dev/ttyUSB0 (or ttyUSB1 for a second unit) and use RTS.
Icom IC-9700 / FM radios with data port: select the radio's USB serial port and use RTS.
—

⬡ HF Channel — Port 1 (300 baud Bell 202 AFSK)

rigctld must be running:
rigctld -m <model> -r <port> -s 19200
CI-V / CAT radios (Xiegu G90, Icom IC-7300 etc.) — use CAT PTT with the radio's USB serial port and Hamlib model.
DigiRig: select /dev/ttyUSB0 and use RTS PTT. Audio goes to plughw:N,0 automatically.
Note: Radios with built-in CM108 USB audio (Xiegu X6100) are not supported — they require PulseAudio which conflicts with reliable AX.25.
—
Audio Levels
Direwolf reports healthy audio at level 50 on its meter. If you see "Audio input level is too high" in the Direwolf log, reduce RX below 40%. If audio is distorted or a carrier with no tones, the input is clipping — reduce RX first.
